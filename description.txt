Приложение будет работать следующим образом.
Список сотрудников. Это будет контейнер, скорее всего, вектор. Одним из методов сотрировки будет сортировка по рейтингу. 
Сотрудник. Имеет ссылку на соответствующую запись в базе данных сотрудников. Сотрудник имеет рейтинг. Класс имеет метод расчета рейтинга. 
Группы сотрудников. Контейнер. Содержит ссылки на сотрудников. Имеет поле - количество разрешенных отдыхающих. Сотрудник может входить в любое количество групп. Подразделение тоже является группой. 
Отрезок времени. Имеет начало и конец типа момента времени. (Момент времени из библиотеки. Или из STL или Boost). 
Части отпуска. Является потомком отрезка. Каждый отрезок принадлежит сотруднику. Часть отпуска может быть  неперемещаемой. 
Заявка. Является потомком отрезка. Каждый сотрудник составляет заявку на желаемые сроки частей отпусков. 
График. Контейнер частей. 

Интерфейс приложения:
Главное меню:
	Список сотрудников;
	Создание графика отпусков;
	Настройки;
	Выход.
Меню списка сотрудников:
	Ручное управление;
	Порядок сотрудников;
	Выход.
Меню графика отпусков:
	Ввод заявок;
	Просмотр;
	Ручное управление;
	Автоматическое формирование;
	Выход. 
Меню настроек:
	Выбор подразделения;
	Подключение к БД;
	Выход.

Модули программы:
	main.cpp;
	menu.[h|c]pp - описываются все меню;
	persons.[h|c]pp
	data_from_sql.[h|c]pp - в модуле описаны объекты, являющиеся оберткой к данным из сервера БД.
